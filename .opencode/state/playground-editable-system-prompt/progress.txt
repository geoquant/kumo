## Codebase Patterns
- lz-string ships bundled types (typings/lz-string.d.ts), no @types package needed
- pnpm --filter @cloudflare/kumo-docs-astro add <pkg> for docs-only deps
- Pre-existing typecheck warnings (unused vars in .astro files) — ignore, not ours
- eslint enforces `_` prefix for unused vars; rename back when wiring into consumers
- PlaygroundContent state lives in `_PlaygroundPage.tsx:508+`; systemPromptText at ~855, editable prompt state at ~872
- URL sync hook at `src/lib/playground/prompt-url-sync.ts`; uses lz-string `compressToEncodedURIComponent` (URL-safe, no extra encoding needed)
- Badge component: `variant` only (primary|secondary|destructive|outline|success|beta), no `size` prop
- PromptEditor at `_PlaygroundPage.tsx:~4497` (now wired; renders in panel A prompt tab)
- PanelContent takes `panelId: PanelId` as required prop; prompt-editor props are optional (panel B doesn't need them)
- ComparisonPanelsProps is the interface for threading props from PlaygroundContent → ComparisonPanels → PanelContent
- Mobile shell (`MobilePlaygroundShell`) also has PanelContent calls — needs `panelId` but NOT prompt-editor props (mobile doesn't wire editable prompt yet)
- Ref indirection pattern for forward-declared callbacks: define `const fooRef = useRef<() => void>(() => {})` early, create stable callback via `useCallback(() => { fooRef.current(); }, [])`, then sync after actual impl: `fooRef.current = realCallback`. See `handleSubmitRef` and `regenerateRef` as examples.
- `startWorkspaceComparison` fires both panels; for Panel-A-only regen, replicate its reset/stream logic without calling `streamPanelB`
- Panel A abort: `abortRef.current?.abort()`, staleness guard: `runIdRef`. Panel B abort: `noPromptAbortRef`

## Task - state-2
- Created `usePromptUrlSync` hook in `src/lib/playground/prompt-url-sync.ts`
- On mount: reads `?promptOverride=` (lz-string decompress) or `?promptRef=` (localStorage lookup) → sets editedSystemPrompt
- On change: debounced (400ms) replaceState; compressed ≤ 1500 chars → `?promptOverride=`, else localStorage + `?promptRef=<hash>`
- On null: removes both params from URL
- Uses FNV-1a 32-bit hash for localStorage keys (deterministic, fast)
- Wired into PlaygroundContent; renamed `_setEditedSystemPrompt` → `setEditedSystemPrompt` since it's now consumed
- Files changed: `src/lib/playground/prompt-url-sync.ts` (new), `src/components/demos/_PlaygroundPage.tsx`
- **Learnings:** `replaceState` preserves scroll position and doesn't trigger navigation; debouncing prevents URL thrash during rapid edits. `compressToEncodedURIComponent` is already URL-safe (no encodeURIComponent needed).
