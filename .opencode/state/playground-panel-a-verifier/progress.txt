## Codebase Patterns
- Keep panel A-only analysis logic in `packages/kumo-docs-astro/src/lib/playground/` and leave panel B request semantics untouched.
- Reuse Kumo grading/validation primitives (`gradeTree`, `gradeComposition`, `validateElement`, `normalizeTree`, `isRenderableTree`, `getUnknownTypes`) instead of rebuilding structural checks from scratch.
- For offline replay, parse raw `/api/chat` SSE by extracting assistant `delta.content` only and ignore reasoning fields before rebuilding JSONL.
- Keep verifier back-pressure tuning in `DEFAULT_PLAYGROUND_VERIFIER_CONFIG`, with separate warn/fail threshold sets passed through `buildPlaygroundVerifierReport({ config })` instead of caller-specific conditionals.
- Gate only panel A preview mount from verifier output: defer render until streaming settles, render warn banners inline, and block fail cases while leaving Tree/JSONL/Grade inspection paths available.
- Reuse compact header popovers plus tab-jump buttons for verifier/debug surfacing, so panel A can expose status + drill-down without changing panel B tabs or adding a new report tab.
- Centralize synthetic verifier trees/SSE builders in `packages/kumo-docs-astro/src/lib/playground/verifier-fixtures.ts` so budget, malformed-structure, and non-renderable cases stay reusable across verifier-focused tests.
