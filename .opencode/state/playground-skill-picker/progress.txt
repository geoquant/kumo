## Codebase Patterns
- `getSkillContents(ids)` returns concatenated skill markdown from `skills-data.generated.ts`
- `skipSystemPrompt` branch in POST /api/chat controls A/B mode system message construction
- `[part1, part2].filter(Boolean).join("\n\n")` pattern for optional content concatenation
- Fetch pattern: `useEffect` + `AbortController` + manual JSON validation (no Zod on client)
- `ReadonlySet<string>` for immutable set state; lazy init with `() => new Set()`
- `lastSubmittedRef` already stores last user message — reuse it for skill apply replay
- `Popover` and `Checkbox` both exported from `@cloudflare/kumo` — compound component pattern
- `PanelHeader` accepts optional `actions?: ReactNode` slot rendered next to label
