# Progress Log
PRD: eval-pipeline-baseline
Started: 2026-02-28

## Codebase Patterns
- CLI eval script: `packages/kumo/scripts/eval-generative.ts` — standalone tsx script, not part of library build
- Eval prompts source of truth: `packages/kumo/src/generative/eval/eval-prompts.ts` — exports `EVAL_PROMPTS`, `EvalPrompt` type
- CLI uses local `EvalPrompt` interface with `name`/`message` fields; shared module uses `id`/`prompt` fields
- Composition graders: `packages/kumo/src/generative/composition-graders.ts` — exports `gradeComposition`, `COMPOSITION_RULE_NAMES`
- Generative barrel: `packages/kumo/src/generative/index.ts` — already exports composition graders
- Structural graders export `RULE_NAMES`, `GradeReport`, `gradeTree`, `parseJsonlToTree`
- Baseline JSON backward compat: read old baselines as `Partial<Baseline>`, check for new fields before comparing
- Skills API: GET `/api/chat/skills` requires `X-Playground-Key` header, returns `{ skills: [{ id, name, description }] }`
- Chat API: POST `/api/chat` accepts optional `skillIds` string[] (max 5) and `X-Playground-Key` header; skills only effective when auth=authenticated
