# Progress Log
PRD: kumo-app-runtime
Started: 2026-03-07

## Codebase Patterns
- Hard rule: keep every task on `geoquant/streaming-ui`; do not branch this work elsewhere.
- New public Kumo subpaths need four touchpoints: source barrel, Vite build entry, `package.json` export, and an import test.
- In `app-runtime/core`, keep path utils, expression resolution, store behavior, and action execution in separate pure modules with focused tests.
- For new runtime semantics, prefer pure snapshot builders (`expandAppSpec`), pure evaluators (`validateFieldsForMode`), and explicit reaction loops (`runWatchers`) over renderer-coupled logic.

- Files changed: `packages/kumo/src/app-runtime/core/repeat.ts`, `packages/kumo/src/app-runtime/core/validation.ts`, `packages/kumo/src/app-runtime/core/watchers.ts`, `packages/kumo/src/app-runtime/core/index.ts`, `packages/kumo/src/app-runtime/index.ts`, `packages/kumo/src/index.ts`, `packages/kumo/tests/app-runtime/repeat.test.ts`, `packages/kumo/tests/app-runtime/validation.test.ts`, `packages/kumo/tests/app-runtime/watchers.test.ts`, `.opencode/state/kumo-app-runtime/prd.json`.
