## Codebase Patterns
- `agents` pins `@modelcontextprotocol/sdk` at `1.23.0` — must match exact version or types diverge (private property clash in `Server`)
- Wrangler `pretypecheck` script generates `types/worker-configuration.d.ts` — file must NOT pre-exist or wrangler errors with "non-Wrangler worker-configuration.d.ts already exists"
- `pnpm-workspace.yaml` uses `packages/*` glob — new packages auto-register, use `catalog:` for shared deps
- McpAgent pattern: extend `McpAgent<Env, State>`, set `server = new McpServer(...)`, implement `async init()`, route via `ClassName.serve('/path', { binding })` in fetch handler
- `agents` bundles MCPClientManager which dynamically imports `ai` — must add `ai` as dependency for wrangler dev to bundle without errors
- Extract DO class to `worker/mcp/index.ts`, re-export from `worker/index.ts` — wrangler types still resolve via `import("../worker/index").ClassName`
- postMessage runtime validation pattern: `isRecord()` guard + reconstruct typed object in `parseIframeMessage()` — avoids domain-level `as` casts on untrusted `event.data`
- Use `.js` extension for local imports (ESM convention with `moduleResolution: "bundler"`)
- `createUIResource` is sync — returns `UIResource` not `Promise`. Metadata keys prefixed `mcpui.dev/ui-`
- Streaming pipeline for iframe apps: `@cloudflare/kumo/streaming` (createJsonlParser, applyPatch) + `@cloudflare/kumo/generative` (UITreeRenderer). `readSSEStream` must be copied (not exported). CSS needs `@import "@cloudflare/kumo/styles"` + `@import "tailwindcss"` + `@source` directives
- React Router + Cloudflare Worker pattern: `reactRouter()` vite plugin + `createRequestHandler(() => import("virtual:react-router/server-build"))` as fallback in Worker fetch. Needs `strip-typegen-imports` plugin for `+types/` stubs, `types/react-router.d.ts` with `export =` for virtual module, `react-router.config.ts` with `ssr: true` + `v8_viteEnvironmentApi: true`
- MCP streamable HTTP requires `Accept: application/json, text/event-stream` — both needed or -32000 error
- MCP proxy pattern: 3-step handshake (initialize → notifications/initialized → tools/call) with `mcp-session-id` header for session affinity. Server may respond with JSON or SSE for any request — parse both.
- Dual tsconfig for kumo-mcp: `tsconfig.json` (worker, no DOM) + `tsconfig.app.json` (iframe app, DOM + JSX). `typecheck` script runs both sequentially.
- postMessage settle pattern: wrap resolve/reject in `settle()` guard to prevent double-resolution from timeout vs message race
- Host→iframe status delegation: McpToolIframe manages `status` as a prop from host. Iframe signals readiness via first non-zero `ui-size-change` → `onStatusChange("pending")`. Host manages the FSM (streaming→pending→applying→completed/cancelled), iframe renders autonomously within that frame
