## Codebase Patterns
- Grader pattern: accumulate violations in `string[]`, build `GradeResult[]` at end, return `GradeReport` with `allPass`
- `walkTree(tree, visitor)` from structural-graders.ts for depth-first traversal — reuse in composition graders
- Types `GradeReport`, `GradeResult`, `WalkVisitor` all exported from structural-graders.ts — import, don't duplicate
- Text heading variants: `heading1`, `heading2`, `heading3` — check via `props.variant`
- Test helper: inline `buildTree(rootKey, elements)` for creating UITree fixtures without JSONL files
- Path aliases (`@/`) resolve at vitest runtime but not in LSP — pre-existing, not a concern
- Simple layout heuristic: ≤12 elements = simple (exempt from complex-only rules like responsive grid requirement)
- Parent-child type check: `walkTree(tree, (el, _depth, parentKey) => { if (parentKey) tree.elements[parentKey].type ... })` — no extra data structure needed
- Sibling grouping: `Map<parentKey, sibling[]>` populated during walkTree, then iterated post-walk for cross-sibling checks (e.g. spacing consistency)
- `allPass` test fixtures must satisfy ALL rules (≥3 elements for content-density, etc.) — update when adding new rules
- Barrel export pattern: `packages/kumo/src/generative/index.ts` re-exports; package.json `exports["./generative"]` maps to dist — add to barrel, not package.json
- System prompt sections array in `buildSystemPrompt()` — order matters; insert new sections at correct position in the array + update JSDoc
- System prompt sections are const string template literals — new section follows same pattern as COMPOSITION_RECIPES, DESIGN_RULES etc.
- `extractExampleJsonl(prompt, title)` test helper: extracts JSONL lines from a named example in the prompt text — reuse for testing any inline example
- Page-level examples follow pattern: `Surface > Stack(gap="lg") > [header, Grid(variant="2up") > [main-col, sidebar-col]]` — consistent two-column structure
- Header-with-actions pattern: `Grid(variant="2up") > [Stack(gap="sm") > [heading, desc], Cluster(justify="end") > [buttons]]` — title left, actions right; satisfies responsive-layout grader for single-column pages
- Tabs pattern in JSONL: `Tabs` element with `tabs` prop (array of `{value, label}`) + `defaultValue` — tab content as sibling Stack elements, not children of Tabs. All panels visible in static tree; runtime handles switching.
- Spacing-consistency gotcha: sibling Stack elements under same parent must have gaps within 1 step (sm↔base OK, sm↔lg NOT OK). Tab content sections sharing a parent with the header must match gap range.
- Regression test pattern: `extractExampleJsonl(prompt, title)` + `parseJsonlToTree()` + `gradeTree()` — data-driven loop over `CARD_EXAMPLES` array with `for...of` + inner `describe` blocks. No need for separate test files per example.
- Eval prompt types: `ExpectedPattern` (union of known layout patterns + "novel") and `RequiredPattern` (structural patterns verifiable by tree inspection) — both exported as union types for type-safe harness code.
