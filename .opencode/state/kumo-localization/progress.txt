# Progress Log
PRD: kumo-localization
Started: 2026-03-05

## Codebase Patterns
- New modules go in `packages/kumo/src/localize/` dir
- Interface pattern: `readonly` fields for immutability, grouped by section with doc comments
- Parameterized translation keys typed as functions `(arg: type) => string` (no MessageFormat runtime)
- `$`-prefixed fields for metadata ($code, $name, $dir) to avoid collision with translation keys
- Pre-existing issues: docs astro typecheck has 1 error (SearchDialog), kumo tests have 70 unhandled rejections in deep-imports.test.ts (Vite worker RPC closing) â€” both pre-existing
- `useSyncExternalStore` pattern for DOM attribute reactivity: lazy observer in `subscribe`, snapshot reads from DOM, server snapshot returns static default
- Files with JSX must use `.tsx` extension (even non-component modules like `index.tsx` in localize/)
- **Component migration pattern**: `import { useLocalize } from "../../localize/index.js"` â†’ `const { term } = useLocalize()` at top of component body â†’ replace hardcoded strings with `term("kebab-key")` calls
- **Translation keys**: kebab-case (e.g., `term("copy-to-clipboard")` not `term("copyToClipboard")`). TypeScript interface uses quoted properties: `readonly "copy-to-clipboard": string`
- **ParameterizedKey type**: Use negation of `string` (`extends string ? never : K`) not function matching (`extends (...args: readonly unknown[]) => string`) â€” avoids contravariance issues with function parameter types
- **Sub-component localization**: Each sub-component calls `useLocalize()` independently â€” no need to thread props from parent. Hook is lightweight (context read + memoized object).
- **Intl.DateTimeFormat for day abbreviations**: Use `getDaysOfWeek(locale)` with `{ weekday: "short" }` + a known Sunday date (Jan 4 2026) to derive localized day names instead of hardcoded arrays
